一文总结计算机网络知识及面试高频考点

Posted on 2019-09-03 |

本文按<<计算机网络>>一书，总结网络重要知识及面试高频考点。重点关注３W中的how与why。这里特别强调下，书中的wireshark实验一定要动手做！

分层

协议

类似信的格式，是通信双方约定俗成的"规则"。

分层原因

类似模块化，优点有：１底层逻辑复用２降低相邻依赖３有利标准化４有利开发和维护（按接口实现新功能、有问题仅关注某层）

封装

类似传递和接收信件，相邻层在协议头或尾加附加信息或去附加信息，从而实现某些功能，当然也可在数据中实现。

分组交换

同样类似传递和接收物资，需分组传输，并且路途中间需"加油站"存储转发。与传统的电话的电路交换需创建并维护线路不同，整体效率是分组更高，原因是电路交换已分配但不用的时间未利用，而分组交换按需分配链路，并共享链路传输。

应用层

进程通信

不同两端系统间（ip与端口）通信的进程通过socket按某数据格式通信，一服务端和一客户端，称为C/S架构（P2P架构和B/S本质仍为C/S），当然复杂系统为两者混合体（QQ用户与服务器通信对应c/s、用户与用户通信对应p2p）。

标准应用层协议

http

客户端由域名解析出ip，并与之３次握手建立tcp连接（更常见的是先与web缓存服务器或代理服务器连接并查找，若无则缓存服务器与原服务器连接并进行后面过程，再响应客户端。），再用请求方法和统一资源定位符URL（简单快速）发送任意类型（Content-Type标记，灵活）资源请求，服务端响应，断开tcp连接（非持续连接，新增持续连接,且为无状态，由Cookie交互），客户端浏览器解析html（B/S及C/S模式）

  • cookie：服务器生成Cookie经Set-Cookie属性通知客户保存Cookie到本地，下次请求时写入Cookie属性，服务器就能分辨各户端，一般用Cookie实现Session。区别见这。

  • https：http over tls/ssl，先通过明文handshake生成密钥，再通过密钥加密http通信。具体见这

ftp

两并行tcp连接：控制、数据连接。支持两种传输方式：文本（ascii）、二进制（binary）。两种传输模式：主动（ftp port）、被动（ftp passive）.区别是传输数据时，前者是服务端连到客户端，而后者相反。前者有安全问题，故多用后者。

电子邮件

提供发送（SMTP/http）、存取（IMAP/POP3/http）邮件服务

dns

客户端通过dns缓存或udp传输协议递归、迭代查询分布式、层次服务器获取域名的ip地址

p2p

传统协议有三大角色：tracker、seeder、downloader。现如今更强，支持DHT网络、NAT穿透、UDP传输。

自定义应用层协议

自定义协议原因

现有标准协议的效率、安全性、扩展性等无法满足现实需求，虽然总的来说最简单可靠的就是使用成熟的https协议，但有时需要自行设计并实现特定程序的协议，实际上自定义协议一般都指应用层协议。优点：非知名协议安全性更高、根据需要扩展。缺点：考虑扩展性、兼容性设计难度高、实现繁琐需自实现序列化和反序列化

序列化与反序列化

由于大小端标准不统一导致字节序分成网络字节序（默认大端，符合直觉）、主机字节序（x86架构是小端）。序列化就是将内存对象转成网络字节流，反序列化相反。常用来持久化、远程传输。

协议设计策略

  • 编码方式：二进制编码（如tcp）、明文文本编码（如http）、混合编码
  • 协议边界：固定边界（带报文长）、模糊边界（特定字节界定报文长）

运输层

运输层概述

类似家庭间信件通信，发送端，运输层将应用层包加上运输层协议首部并移动到网络层，接收端，运输层获取网络层传来的包并去掉运输层协议首部并传给应用层，总之，运输层协议仅工作在端系统，不管包在网络核心中如何移动，相反，网络核心中的路由器既不处理也不识别运输层加在应用层报文的任何信息。（细细体会是不是和软件开发中的模块化思想一样：））。所以说运输层协议为不同主机上的进程提供逻辑通信功能。

多路复用和多路分解

同样类似家庭信件通信，发送端，运输层从不同套接字收集数据块（应用层包），并封装上运输层首部生成运输层包并传递到网络层称为多路复用，接收端，运输层将包剥去报文首部并能将包交付到正确的套接字称为多路分解。这其中的关键是ip地址与端口标识套接字（注意运输层报文中仅有端口而无ip，ip会在网络层中添加）。udp套接字由目的ip与目的端口二元组标识，tcp套接字由源ip、源端口及目的ip、目的端口四元组标识。换言之，两个相同目的ip与目的端口的udp报文会被分解到相同套接字，而两个相同目的ip与目的端口但源ip与源端口不同的tcp报文会被定位到两个不同套接字。故tcp连接与通信是不同的两个套接字。

udp

udp想对于网络层协议仅增加复用/分解功能和有限的差错检测功能（丢弃受损包或仍正常传输但给应用程序警告）

udp提供差错检测的原因

  • 无法确保源和目的间所有链路都提供差错检测
  • 无法确保链路中路由器的内存是否出错

tcp

tcp协议要实现可靠传输、流量控制、拥塞控制三大功能，因而也是最复杂的标准协议之一。具体说来，tcp为了保证可靠性，需实现功能：

  • 差错检测：校验和（相加、进位补后、取反得校验和）
  • 序列号与确认应答：当服务端接收到每个字节都编号（序列号）的数据后，都会通过带有确认序列号的ACK报文应答，告知客户端接收到哪些数据与下次从哪开始发送。
  • 连接管理：采用三次握手、四次挥手，保证传输信道可靠，是可靠传输的前提。
  • 重传机制：ARQ（Automatic Repeat-reQuest即自动重传）协议包括停止等待ARQ与连续ARQ协议，前者效率不高，后者包括GBN（Go-Back-N即回退N步）与SR（Selective Repeat即选择重传）协议。ARQ协议会连续发送一组数据包，再等待这些数据包的ACK，发送方每收到一个确认，就把发送窗口向前滑动一个分组位置，若长时间未收到以致超时（动态计算超时时间）则超时重传。接收方对按序到达的最后一个分组发送累积确认，表示到该分组为止的所有分组都已正确收到。
  • 流量控制：通过滑动窗口确保接收方能及时处理所接收的数据。详细说明请参考文章。发送窗口和接收窗口都分成４个部分，如图所示：滑动窗口遵循３个规则：
    • 已发送过数据未收到确认前，都暂时保留以便超时重传使用
    • 发送方收到接收方确认时，发送窗口才向前滑动几个序号
    • 发送方发送的数据经过一段时间未收到确认（超时计时器控制）就回退N步选择重传这部分数据。
  • 拥塞控制：慢开始、拥塞避免、快重传和快恢复。 常被概括为加性增、乘性减（Additive-Increase,Multiplicative-Decrease,AIMD）。拥塞窗口演化如图：

注意：流量控制是指：控制发送速率使接收方来得及接收，防止丢失分组；拥塞控制是指：防止太多数据注入网络，避免网络负载过大。类似交通运输，前者类似于控制车速，后者类似于控制车流防止拥堵。这两者不要搞混。

网络层

转发和路由选择

类似驾车旅行，转发是指将数据包从输入链接接口传输到输出链接接口的路由器本地操作，路由选择是指通过路由选择算法确定数据包从源到目标的端到端的网络范围的过程。

虚电路和数据报

对比 虚电路 数据报
建立连接 有 无
目标地址 每个分组仅连接建立时使用虚电路号 每个分组都有目的站的全地址
路由选择 建立连接时进行且所有分组路由相同 每个分组独立选择路由
故障 所有过故障路由器的虚电路不能工作 故障路由器可能丢失分组，路由可能改变
分组顺序 总是按发送顺序到达目的站 到达目的站时可能不按发送顺序
差错处理 由通信子网负责 由主机负责
流量控制 由通信子网负责 由主机负责

ip协议

ip数据报分片

由于底层链路层受MTU（Maximum Transmission Unit即最大传送单元）限制，且发送方和目的地路径上的每段链路可能使用不同MTU的链路协议，故ip数据报需分片。有分片就有组装，为了简单将数据报重组工作放在端系统，而不是网络路由器（想想就复杂），于是ip数据报需要标识、标志和片偏移字段实现重组，但也带来复杂性和安全性（偏移量均不为0的DoS攻击、交跌ip片导致无法重组）
不过IPv6废止分片，也就没有其带来的复杂性和安全性问题。

IPv4编址

为了不使用ip完全匹配的转发策略而使得转发表太长，引入子网寻址（子网掩码），进而将其概念一般化也就是CIDR（Classless Interdomain Routing即无类别域间路由选择），不过在此之前使用分类编址的编址方案（A、B、C、D类） 。由于IPv4的32位IP地址有限，使用以下方法或协议充分利用ip地址：

  • 从ICANN处获取ISP的IP地址，并从ISP获取一个组织的子网ip
  • 通过管理员手动分配或即插即用的DHCP（Dynamic Host Configuration即动态主机分配）协议分配子网中的主机ip
  • 通过NAT（Network Address Translation即网络地址转换）解决小型办公室子网的增长需求。但有人反对NAT：
    • 端口号用于进程编址而非主机编址
    • 路由器仅因当处理高达地三层的分组
    • NAT违反端到端原则
    • 妨碍P2P应用，本质是无法通过TCP通信，不过可通过NAT穿越解决，常用UPnP提供。
      当然最根本方法还是增加可分配的IP地址数即IPv6，高达2^128个IP地址，同时也能兼容IPv4协议（兼容性也是软件开发中主要成功因素之一）。

ICMP

由于IP协议非可靠传输，故通过ICMP测试并通知网络差错情况。

IPv6

随着IP地址耗尽，从IPv4迁移到IPv6显得迫在眉睫，尽管IPv6兼容IPv4，但IPv4却不能处理IPv6，可采用的方法有：
宣布某个标志日届时所有机器关机并全部从IPv4升级到IPv6
双栈方法1通过DNS查询是否是IPv6使能或仅IPv4使能，在使用IPv4或IPv6通信２通过将整个IPv6数据报放到IPv4数据报中的数据字段中，即建隧道实现双栈
IPsec协议兼容IPv4与IPv6，同时路由器和协议均不变，只需要IPsec在两台主机中可用。

路由选择算法

按照是全局还是分散可将路由选择算法分为：全局路由选择算法、分散路由选择算法。实践中常分别被称为链路状态算法和距离向量算法。
前者使用Dijkstra算法实现，后者通过Bellman-Ford方程计算值，值得一提的是在遇到路由选择环路的情况时，可使用毒性逆转技术加以避免。实际中由于规模和管理自治的原因将路由器组织进自治系统，在系统内在使用前面的两种算法。

因特网中的路由选择

历史上两个路由选择协议曾被广泛用于自治系统内的路由选择：RIP（Routing Information Protocol即路由选择信息协议）和OSPF（Open Shortest Path First即开放最短路优先）。自治系统间的路由选择常用BGP（Broder Gateway Protocol即边界网关协议）

广播和多播路由选择

从一个结点到其他所有结点交付分组叫广播路由选择，一个结点到其他真结点的子集（部分结点）发送分组叫多播路由选择。不细说了。
